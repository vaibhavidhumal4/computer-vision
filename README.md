# FoodLens — AI Food Recognition App

> Identify 101 food dishes from photos using MobileNetV3 fine-tuned on Food-101.
> Built with FastAPI + PyTorch + a clean HTML/JS frontend. Fully Dockerized.

---

|---|---|---|

## Quick Start (without training)

The app runs in demo mode using ImageNet pretrained weights.
Predictions work but accuracy is lower until you train.

### 1. Install dependencies
```bash
pip install -r requirements.txt
# For GPU (RTX 3050):
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
```

### 2. Run the backend
```bash
cd backend
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 3. Open the frontend
Open `frontend/index.html` in your browser, or visit:
```
http://localhost:8000
```

## Train the Model (Recommended for best accuracy)

Training fine-tunes MobileNetV3-Large on the Food-101 dataset (~75–80% top-1 accuracy).

```bash
# From the project root:
python train.py
```

What happens:
- Food-101 dataset (~5GB) downloads automatically to `./data/`
- Trains for 10 epochs (~2–3 hours on your RTX 3050)
- Best model saved to `model/food_model.pth`
- The API auto-loads this file on startup

Expected accuracy milestones:
- Epoch 1: ~30%
- Epoch 5: ~60%
- Epoch 10: ~75–80%

## Docker Deployment

### Build & run (CPU mode)
```bash
docker compose up --build
```
App available at: http://localhost:8000

### Enable GPU in Docker (RTX 3050)

1. Install NVIDIA Container Toolkit:
```bash
# Windows: use Docker Desktop with WSL2 + NVIDIA drivers
# Linux:
distribution=$(. /etc/os-release;echo $ID$VERSION_ID)
curl -s -L https://nvidia.github.io/nvidia-docker/gpgkey | sudo apt-key add -
curl -s -L https://nvidia.github.io/nvidia-docker/$distribution/nvidia-docker.list \
    | sudo tee /etc/apt/sources.list.d/nvidia-docker.list
sudo apt-get update && sudo apt-get install -y nvidia-docker2
sudo systemctl restart docker
```

2. Uncomment the GPU section in `docker-compose.yml`:
```yaml
deploy:
  resources:
    reservations:
      devices:
        - driver: nvidia
          count: 1
          capabilities: [gpu]
```

3. Rebuild:
```bash
docker compose up --build
```

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| GET | `/` | Serve frontend |
| GET | `/health` | Model status, device, class count |
| POST | `/predict` | Upload image → top-5 predictions |
| GET | `/classes` | List all 101 food categories |

### Example API call (curl)
```bash
curl -X POST http://localhost:8000/predict \
  -F "file=@pizza.jpg"
```

### Example response
```json
{
  "predictions": [
    {
      "label": "Pizza",
      "confidence": 94.2,
      "calories_per_serving": 285,

    },
    ...
  ],
  "inference_time_ms": 42.3,
  "device": "cuda",
  "image_size": "640x480"
}
```

## Extending the Project

- **Add RBAC**: Add FastAPI JWT middleware with admin/user roles
- **Kubernetes**: Use the provided Dockerfile with a local Minikube cluster
- **Nutrition API**: Replace hardcoded calories with Edamam or Nutritionix API
- **Image history**: Add SQLite to log past predictions per user
- **Batch prediction**: Add a `/predict/batch` endpoint for multiple images
